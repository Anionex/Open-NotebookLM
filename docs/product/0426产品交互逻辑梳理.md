# 说明

- 仅支持推送、不再单独修改文档

- 主文+暂存区

- 引入焦点机制
  - 文档左侧边栏凸显

- 任何时刻，一个对话有且仅有一个"活跃文档"。【表示“ai也在看/可操作”】

- 历史管理与文档变更感知。
  - 增量版本快照+变更日志
  
  - AI 需要知文档变更：参考实现方式（每次发送消息，系统在 context 里**额外注入"自上次本对话发消息以来的文档变更摘要"**）
  
  - 用户可撤销某次ai推送带来的修改 or 自身的修改
  
  - 实现注意：
  
    系统提示词，每一次ai先感知用户对文档做了哪些修改
  
- 产出：每份梳理文档勾选后，下方展开"包含范围"

  - 转化工作台，左侧只有产出文档（此时认为用户已对此确认）、右侧是ppt预览与生成修改工作台


# thinkflow产品交互逻辑

## 0. 总体架构

**三栏布局**：

- **左栏（可拖拽）**：三个 tab —— 对话 / 素材 / 产出
- **中栏（自适应）**：对话区
- **右栏（可拖拽）**：文档区

**核心实体关系**：

- 工作空间包含：素材、对话、文档（梳理/产出）、产出物
- 对话有自己的来源集合（指向素材或其他梳理文档）
- 对话任意时刻有且仅有一个**活跃文档**
- 每份文档有自己的焦点状态、版本历史

------

## 1. 主页与工作空间

### 1.1 进入产品

打开应用 → 主页。看到所有工作空间的卡片网格，每张卡片显示名称、描述、对话数、文档数、上次编辑时间。最后一张是"➕ 新建工作空间"。

### 1.2 新建工作空间

点击"➕ 新建工作空间" → 弹 modal：

- **名称**（必填）
- **描述**（可选）
- 取消 / 创建

点创建 → 后端建空间，初始化一份空白梳理文档（标题"新文档"），创建一个"对话 1"，进入工作空间。

### 1.3 进入已有工作空间

点击卡片 → 进入。默认打开最近一次访问的对话和文档，焦点恢复到上次状态。

### 1.4 工作空间内的全局操作

顶栏：logo（点击回主页）、工作空间名（可点编辑）、🕐 历史、⚙️ 设置。

------

## 2. 左栏：素材 tab

### 2.1 素材列表

每条素材显示：图标、名称、大小。悬停出现操作按钮：✕ 删除。

点击素材本体 → 弹抽屉预览原文（PDF/图片/文本），不影响其他状态。

**注意**：素材本身没有"激活/未激活"开关。素材是否进入 AI 上下文，完全由对话的"来源行"决定。

### 2.2 添加素材

素材列表底部有"➕ 上传"按钮。点击 → 弹文件选择器 → 上传 → 后端解析（PDF 提文字、图片走 OCR 或视觉模型）→ 出现在列表里。

也支持拖拽文件到素材区上传。后续支持粘贴 URL（弹一个小 input，确认后后端抓取）。

------

## 3. 左栏：对话 tab

### 3.1 对话列表

每条对话显示：标题、消息数、最后时间、当前激活的来源数。悬停出现：✎ 重命名、✕ 删除。

点击对话 → 中栏切到这个对话，恢复其状态（消息、来源行、活跃文档）。

### 3.2 新建对话

点列表上方"➕ 新建对话" → 弹 modal：

```
新建对话

标题：[___________________]

📎 这次对话基于哪些来源？

  ─── 素材 ───
  ☐ 行业报告.pdf
  ☐ 竞品A手册.pdf
  ☐ 调研数据.xlsx
  
  ─── 梳理文档 ───
  ☐ 竞品分析

💡 之后发送消息时也可以临时加入

[取消]  [开始对话]
```

**默认全不勾选**——让用户主动选这次需要哪些。点开始 → 创建对话 → 中栏切到新对话。新对话默认活跃文档是右栏当前展示的那个文档。

------

## 4. 中栏：对话区

### 4.1 顶部：来源行

每个对话顶部一行：

```
📎 来源 (3) ▾                           [+ 添加]
```

**折叠态**：只显示数量。`(0)` 时也显示但是淡色。

**展开态**（点 ▾）：

```
📎 来源 (3) ▴                           [+ 添加]
─────────────────────────────────────────
  📕 行业报告.pdf                    [✕]
  📕 竞品A手册.pdf                   [✕]
  📊 调研数据.xlsx                   [✕]
─────────────────────────────────────────
```

**操作**：

- 点 ▾/▴ 切换折叠/展开
- 点单条的 ✕ 移除该来源（仅本对话临时移除，不删除素材）
- 点 [+ 添加] 弹来源选择 popover：

```
+ 添加来源

─── 素材 ───
☐ 行业报告.pdf  (已添加)
☐ 竞品A手册.pdf (已添加)
☑ 调研数据.xlsx
☐ 旧文档归档.pdf

─── 梳理文档 ───
☐ 竞品分析
☐ Q4 路演 PPT (产出)

[取消]  [添加]
```

已添加的项灰色标注，避免重复。点添加 → 新来源进入对话，下次发送时纳入 context。

### 4.2 消息流

**用户消息**：右对齐，蓝色气泡。

 **AI 消息**：左对齐，浅色气泡。

每条悬停出现操作按钮：⟩ 推送、📋 复制、☐ 多选。 

**用户消息**：也可推送。

**系统标记**：当用户切换活跃文档时，对话流中插入一条系统标记（见 5.4），灰色细条样式，不可编辑不可删除。



### 4.3 多选

点任意消息上的 ☐ → 进入多选模式 → 该消息 checkbox 勾上。继续点其他消息可加选。

进入多选模式后，中栏底部出现操作 bar：

```
✓ 已选 3 条                         [⟩ 推送]  [取消]
```

点 [⟩ 推送] → 弹推送 popover（源内容是多条消息按时间顺序拼接）。点 [取消] 退出多选模式。

### 4.4 输入框

底部输入框 + 发送按钮（↑）。回车发送，Shift+回车换行。

发送时系统组装 context：

```
[系统提示词]
[活跃文档完整 markdown 内容]
[活跃文档当前焦点描述：例如 "焦点：§2 + §3"]
[来源 1 全文]
[来源 2 全文]
...
[变更摘要：自上次本对话发送以来，活跃文档发生的变更]
[对话历史消息]
[用户新消息]
```

Note：文档变更摘要由系统从变更日志生成（见 7.3），让 AI 感知用户在 chat 之外做的修改。

------

## 5. 右栏：文档区

### 5.1 顶部：文档标签栏

横向标签条，每个 tab 是一份文档：

```
[📄 竞品分析 ●] [📊 Q4 PPT] [+ 新建]
```

- **当前活跃文档**：实色高亮 + 左侧有一个圆点 ● 标记
- **其他文档**：淡色，点击可切换显示

> **重要**：文档"显示"和"活跃"是两个状态。
>
> - 显示 = 用户当前看的是哪个 tab
> - 活跃 = 当前对话的 AI 上下文绑定的是哪个文档 两者通常一致（点击 tab 同时切显示和活跃），但通过下方的"活跃文档"标识让用户清晰感知。

### 5.2 顶部：焦点条

文档标签栏下方一行：

```
📍 焦点：§2 核心功能 + §3 定价                    [✕ 回到全文]
```

**焦点描述自动生成**：

- 单段："📍 焦点：§2 核心功能"
- 多段："📍 焦点：§2 核心功能 + §3 定价"
- 全文："📍 焦点：全文"
- 暂存区："📍 焦点：暂存区（3 条）"

[✕] 一键清空所有选中，回到全文焦点。

### 5.3 顶部：活跃文档标识【表示“ai也在看/可操作”】

焦点条下方再加一行（仅在右栏当前显示文档 ≠ 当前对话活跃文档时出现）：

```
⚠️ 当前显示的是「Q4 PPT」，对话活跃文档是「竞品分析」
[切换活跃为本文档]                                     
```

**正常情况**（显示=活跃）：这一行不出现，节省视觉空间。

**异常情况**（显示≠活跃）：用户点了别的 tab 但没切活跃，提示用户"AI 不在看这份"，并提供一键切换按钮。

### 5.4 切换活跃文档

**切换方式 1**：点击非活跃文档的 tab。直接切活跃 + 切显示。

**切换方式 2**：在显示≠活跃的提示行点击 [切换活跃为本文档]。

**切换发生时**：

1. 当前对话的活跃文档变更
2. 中栏对话流插入一条系统标记：

```
─────────────────────────────────
📄 切换活跃文档：竞品分析 → Q4 PPT
─────────────────────────────────
```

1. 焦点条恢复到新文档之前的焦点状态
2. 下一条用户消息发送时，context 基于新活跃文档组装

**没有"撤销切换"**——以当前活跃文档为准。如果用户切错了，再切回去就行（会再插一条系统标记，对话流会显示两条切换记录，这是对的——它如实反映了用户的操作）。

### 5.5 文档主体：左侧边栏

文档每个章节（## 二级标题 及其内容）作为一个**自然块**。每个块左侧有一条 4-6px 宽的边栏，贯穿整个块。

**视觉状态**：

- **未选中**：浅灰 `#e5e4e1`，几乎隐形
- **悬停**：中蓝 `#93c5fd`，提示可点
- **选中（在焦点中）**：实蓝 `#2563eb`，并且块内容区有淡蓝背景 `#eff6ff`

**交互**：

- 点击边栏任意位置 → 切换该块的焦点状态
- 没有 Shift/Cmd 多选——单击就是 toggle，多次点不同块自动实现多段
- 点击焦点条上的 [✕] 一键清空

**初版只支持二级标题（##）作为最小单元**。三级以下作为二级块的内容，不可单独选择。

### 5.6 文档主体：内容编辑

**默认渲染态**：markdown 渲染后的内容，可读不可编辑。

**编辑态**：双击文档任意位置 → 进入编辑态，光标定位到双击位置，可正常打字编辑。失去焦点（点其他地方）退出编辑态，重新渲染。

或者文档右上角有 [✎ 编辑] / [✓ 完成] 切换按钮（更明确）。

**用户编辑后的处理**：

- 编辑发生时，本地立即更新文档内容
- **停顿 3 秒**后自动打快照（见 7.1）+ 写一条变更日志（见 7.2）
- 用户主动 [✓ 完成] 也立即触发快照
- 不阻塞用户，所有保存是后台异步

### 5.7 暂存区

文档底部的特殊区域：

```
─────────────────────────────────
▼ 📥 暂存区 (3)
─────────────────────────────────
```

**默认折叠**，标题栏显示数字徽章。

**展开后**：每条暂存项是一个独立的块，有自己的左侧边栏（同样可点选作为焦点）。每条显示：

```
▎ [来源: 对话1 · 14:30]                  [📤] [✕]
  推送的实际内容...
  推送的实际内容...
```

- [📤] 整理进正文：弹一个轻 popover，选目标章节 + 处理方式 + 指令，本质是二次推送。
- [✕] 删除：移除该条。

**暂存区可作为焦点**：

- 点单条暂存项的边栏 → 焦点 = 那一条（焦点条："📍 焦点：暂存区第 1 条"）
- 点暂存区折叠条本身的边栏（折叠条的左边缘充当边栏）→ 焦点 = 整个暂存区

## 6. 推送：AI 修改文档的唯一入口

**核心规则**：AI 永远不会主动修改文档。所有 AI 对文档的修改都通过用户显式触发的"推送"动作。

### 6.1 触发推送

**单条推送**：点 AI 或用户消息的 ⟩ 按钮。

**多选推送**：多选模式下，底部 bar 的 [⟩ 推送]。

### 6.2 推送 popover

```
⟩ 推送到文档

📍 目标位置：[当前焦点：§2 核心功能 ▾]
   下拉打开：
   ─── 推荐 ───
   📍 当前焦点：§2 核心功能（默认）
   ─── 现有章节 ───
   §1 市场概况
   §2 核心功能 ✓
   §3 定价
   §4 风险
   ─── 其他 ───
   + 新建章节
   📥 暂存区
   📄 文档末尾

🔧 处理方式：
   ◉ AI 根据指令整理后追加到目标位置
   ○ 原文直接追加到目标位置
   ○ AI 根据指令融合到目标位置（修改/替换现有内容）

✏️ 指令（可选）：
   [textarea: 例：只取关于市场规模的部分；语言浓缩一些]

[取消]  [推送 ⟩]
```

**动态启用规则**：

- 选了"+ 新建章节" / "📥 暂存区" / "📄 文档末尾" → 第三种处理方式（融合）灰掉
- 选了"原文直接追加" → 指令框灰掉

**目标位置默认值**：当前焦点（如果焦点是全文，默认值变为"📄 文档末尾"）。

### 6.3 推送执行

点 [推送 ⟩] →

1. 关闭 popover
2. **打快照**（推送前的文档状态）
3. 后端组装 context：

```
[系统提示词：你正在执行推送操作]
[源消息内容（单条或多条按时间拼接）]
[当前对话最近 5 轮历史消息]
[活跃文档完整内容]
[目标位置周围内容（如果是现有章节，把整节内容连同前后章节标题一起给 AI）]
[处理方式]
[用户指令]
```

1. LLM 流式生成结果
2. 系统应用结果到文档对应位置
3. 文档区目标位置高亮闪烁一下作为提示
4. **再打一次快照**（推送后的状态）
5. 写一条变更日志
6. 推送完成后，**焦点自动跟到目标位置**
7. 中栏对话流：被推送的消息打上"已推 ✓"标记

### 6.4 推送失败

后端报错或 LLM 超时 → 不修改文档，弹 toast "推送失败，请重试"。已经打的"推送前快照"标记为废弃（不进版本历史）。

## 7. 历史管理

### 7.1 版本快照

每份文档维护版本快照列表：

```
{
  id: 'ver-001',
  doc_id: 'doc-1',
  content: '<完整 markdown>',
  timestamp: '2026-04-27 14:30:12',
  author: 'user' | 'ai-push' | 'ai-edit',
  source: {
    type: 'push' | 'manual-edit' | 'output-gen' | 'restore',
    conv_id: 'conv-1',
    msg_ids: ['msg-3'],
    instruction: '只取关于市场规模的部分'
  },
  description: 'AI 在 §2 整理后追加（约 120 字）'
}
```

**打快照时机**：

- 推送前 + 推送后
- 用户编辑停顿 3 秒后
- 用户主动 [✓ 完成] 编辑
- 文档创建时（初始版本）
- 用户从历史面板恢复版本时（恢复后的状态）

**保留策略**：每文档保留最近 50 个版本，更老的归档（不主动删，但前端不展示）。

### 7.2 变更日志

更细粒度的事件流，每个动作记一行：

```
{
  timestamp: '14:30:12',
  doc_id: 'doc-1',
  type: 'user-edit' | 'ai-push' | 'switch-active' | ...,
  summary: '用户在 §2 修改了 ~50 字',
  related_conv: 'conv-1'  // 关联到哪个对话（如果有）
}
```

**变更日志保留最近 100 条**或 7 天。

### 7.3 AI 感知用户修改

每次发送消息组装 context 时，系统从变更日志取出**自本对话上次发送以来发生的、关于活跃文档的变更**，生成摘要：

```
[文档变更摘要]
自上次对话以来，「竞品分析」发生了以下修改：
- 14:30 用户编辑了 §2（约 50 字调整）
- 14:35 用户在 §3 新增了一段（约 80 字）
请在回答时注意这些变化已反映在当前文档中。
```

放进 context。AI 据此回答。系统提示词里有指令："文档可能在你不参与时被用户修改，请关注'文档变更摘要'部分。如果用户提及'我刚改的'之类，参考摘要定位。"

### 7.4 撤销与重做

**Cmd+Z 撤销**：版本指针后退一步，文档内容恢复到上一快照。 **Cmd+Shift+Z 重做**：指针前进。

撤销不删除版本——指针可在版本链上前后移动。**只有当用户撤销后又做了新操作，从该位置开始的"未来分支"才丢弃**。

撤销范围 = 当前显示的文档（不跨文档不跨对话）。

### 7.5 版本历史面板

文档区右上角有 [↺ 历史] 按钮。点击 → 右侧滑出面板：

```
版本历史 · 竞品分析

● 当前版本
  14:40 · AI 推送融合到 §2
  ├ 来自 对话 1 / msg-12
  ├ 指令: "整合数据"

○ 14:35 · 用户编辑 §3
  [恢复到此版本]

○ 14:32 · AI 推送新建 §3
  ├ 来自 对话 1 / msg-7
  [恢复到此版本]

○ 14:30 · 用户编辑 §2
  [恢复到此版本]

○ 14:00 · 文档创建
  [恢复到此版本]
```

每个版本条目显示：时间、操作描述、来源信息（关联对话/消息/指令）。当前版本顶部，越往下越老。

**点 [恢复到此版本]** →

1. 弹确认 toast："恢复后当前未保存的修改会保留在最新版本中，确认？"
2. 用户确认 → 文档内容覆盖为该版本快照
3. 同时**新建一个快照**（描述："恢复到 14:32 版本"）作为当前版本
4. 不删除中间版本

**点关联消息** → 跳到中栏对话中的对应消息位置（高亮闪烁）。

# 8. 产出文档：以 PPT 为例

## 8.1 创建产出文档

文档标签栏的 [+ 新建] → 弹文档类型选择：

```

**关于梳理文档作为来源的子选项**：

==每份梳理文档勾选后，下方展开"包含范围"==：

- ☑ **主文**（默认勾上）：文档主体内容
- ☐ **暂存区（N 条）**：默认不勾。如果暂存区为空，这一项灰掉提示"暂存区为空"

**为什么主文默认勾、暂存区默认不勾**：主文是用户已确认整理的内容，是产出的主要原料；暂存区是未消化的零散条目，通常不希望直接进入产出生成。但保留勾选给用户兜底——如果用户觉得某些零散内容也该被纳入，可以打开。

**素材**：直接勾选，无子选项（素材是整体）。

点 [创建并生成初版] →

1. 后端读取所选来源的指定范围内容
2. LLM 基于来源 + 设定生成产出文档（markdown 形式）
3. 文档结构按下方模板生成：

  核心内容：
  - 竞品 A 内容+社区路线

	竞品 A 高端化苗头：具体内容....

```

1. 落成新文档，自动切到这份文档（设为活跃 + 显示）
2. 焦点 = 全文

## 8.3 在产出文档上协作（产出工作台之前）

如果用户从产出工作台返回了主工作台编辑产出文档（见 8.7），所有交互（对话、推送、焦点、暂存区、历史）和梳理文档完全一致。区别只有：

**(1) 文档顶部的可点击设定标签**：

```
[📎 来源 (1)]  [🎨 商务简洁]  [👥 产品团队]  [🎯 15 分钟分享]
```

每个标签点击 → 弹小 popover 内联编辑该项。修改后立即更新文档头部对应字段。

**(2) 文档顶部右侧 [🚀 生成 PPT] 按钮**：再次进入产出工作台。

## 8.4 进入产出工作台

产出文档创建成功后，**页面丝滑过渡到产出工作台**（视觉上是工作空间内的另一个"模式"，不是新开窗口）。

顶栏新增面包屑：

```
[ThinkFlow] / 工作空间名 / Q4 路演 PPT · 产出工作台    [← 返回工作空间]
```

**布局变化**：

```
┌─ 左栏（产出文档区）────┬─ 右栏（PPT 预览/编辑区）─────┐
│ # Q4 路演 PPT          │  状态：待确认大纲             │
│                        │                                │
│ ## 📎 来源              │  [大纲预览]                   │
│ ## 🎨 风格              │   - 第 1-3 页：市场全景        │
│ ## 👥 受众              │   - 第 4-7 页：差异化路线      │
│ ## 🎯 目标              │   ...                          │
│ ---                     │                                │
│ ## 内容大纲             │  [✓ 确认并生成 PPT]           │
│ - 目录                  │  [← 返回修改大纲]  
│
│ - 第一章 ...            │    [ppt生成后逐页修改]        │
│ - 第二章 ...            │                                │
│                         │                                │
│ ## 每章其他素材         │                                │
│ ## 附注                 │                                │
└─────────────────────────┴────────────────────────────────┘
```

**核心变化**：

- 中栏对话区**移除**——产出工作台不在这里聊天
- 左栏 = 产出文档（可直接编辑）
- 右栏 = PPT 流程区（按状态切换内容）
- 顶栏多了"返回工作空间"按钮

**产出工作台的工作流是有阶段的**：

1. **阶段 A：确认大纲**（默认进入时）
2. **阶段 B：AI 生成中**
3. **阶段 C：PPT 已生成，逐页修改**

每个阶段右栏显示不同内容。左栏的产出文档**始终可编辑**——用户可以随时回到大纲层修改。

## 8.5 阶段 A：确认大纲

进入产出工作台默认是这个阶段。

**右栏内容**：

```
状态：📋 待确认大纲

基于左侧产出文档，PPT 大致结构如下：

─── 大纲预览 ───
1. 封面页
2. 目录
3-5. 市场全景（第一章）
6-9. 差异化路线（第二章）
10-14. 核心策略对比（第三章）
15-17. 趋势与启示（第四章）
18. 结语
─────────────

风格：商务简洁
受众：产品团队
预计 18 页

[✓ 确认并生成 PPT]
[💡 让 AI 优化大纲]
```

**用户操作**：

- **直接编辑左栏文档**：改章节、加附注、调风格——右栏的大纲预览实时同步
- **点 [💡 让 AI 优化大纲]**：弹小 popover 输入指令（"压缩到 12 页"、"加一页风险讨论"等）→ AI 修改左栏文档 → 右栏预览更新
- **点 [✓ 确认并生成 PPT]**：进入阶段 B

> **没有对话区，但 [💡 让 AI 优化大纲] 提供了一次性的 AI 干预入口**——这是产出工作台的简化设计：用户主要靠手动编辑左栏文档，遇到需要 AI 协助时调用单次操作，不再开启长对话。

## 8.6 阶段 B：AI 生成中

点 [✓ 确认并生成 PPT] →

1. 后端组装示例：

```
[系统提示词：生成 PPT 内容]
[产出文档完整内容]
[所有引用的来源全文]
```

1. LLM 流式输出每一页的具体内容
2. 右栏切到生成进度状态：

```
状态：✦ AI 正在生成 PPT...

进度：第 6 页 / 共 18 页
├─ ✓ 第 1 页 · 封面
├─ ✓ 第 2 页 · 目录
├─ ✓ 第 3 页 · 市场规模概览
├─ ✓ 第 4 页 · 增速对比
├─ ✓ 第 5 页 · 三家竞品概况
├─ ⠋ 第 6 页 · 差异化路线引入
└─ ...

[取消生成]
```

1. **生成期间左栏文档锁定为只读**，避免用户改动导致结果与文档不一致
2. 生成完成 → 右栏切到阶段 C

## 8.7 阶段 C：逐页修改 【示例】

```
状态：✓ PPT 已生成（18 页）

[预览面板]
┌─────────┬──────────────────────────────┐
│ 缩略图   │  当前页：第 6 页              │
│         │                                │
│ □ 1     │  [大幅渲染当前选中页]         │
│ □ 2     │                                │
│ ▣ 3     │  标题：差异化路线引入          │
│ □ 4     │  正文：...                     │
│ □ 5     │                                │
│ ▶ 6     │  ─────────────────             │
│ □ 7     │  💬 对这一页提出修改意见：     │
│ ...     │  [textarea]                    │
└─────────┴──────────────────────────────┘

[⬇ 下载 .pptx]  [🔄 重新生成全部]  [← 返回修改大纲]
```

**右栏内容结构**：

- **左侧缩略图栏**：所有页的预览图，竖向滚动，当前选中页高亮
- **中央大图**：选中页的渲染预览
- **底部 prompt 框**：针对当前页的修改输入框

**用户操作**：

**(1) 选页**：点缩略图 → 切换中央大图

**(2) 单页 prompt 修改**：底部输入框输入指令 → 点 [应用修改] →

- 后端只针对这一页重新生成（context = 该页当前内容 + 用户指令 + 产出文档头部设定）
- 中央大图实时刷新
- 缩略图同步更新

例：

```
💬 对第 6 页提出修改意见：
"把这页的对比图换成竖向柱状图，配色用蓝色系"
[应用修改]  [取消]
```

**(3) 直接编辑当前页文字**：双击中央大图的某个文字区域 → 进入编辑态 → 改文字 → 失去焦点保存。这是最轻的修改方式，绕过 LLM。

**(4) 调整页顺序**：缩略图栏支持拖拽排序

**(5) 删除页**：缩略图悬停出现 ✕

**(6) 插入新页**：缩略图之间出现"+ 插入"分隔线，点击后输入新页主题，AI 生成

**(7) 全局操作**：

- [⬇ 下载 .pptx]：导出文件，归档到工作空间产出列表
- [🔄 重新生成全部]：弹确认 → 回到阶段 B 重新生成（保留产出文档不变）
- [← 返回修改大纲]：回到阶段 A，PPT 内容保留但状态回到大纲层。再次点 [✓ 确认并生成] 时弹确认："已有生成结果，重新生成会覆盖。继续？"

## 8.8 返回工作空间

顶栏点 [← 返回工作空间] →

- 回到主工作空间布局（三栏：对话/中栏对话/右栏文档）
- 当前活跃文档恢复为产出文档（用户可继续在那聊和推送）
- 产出工作台的 PPT 状态保留（再次从产出文档点 [🚀 生成 PPT] 回来时直接进入阶段 C，不需要重新生成）

## 8.9 历史与版本

**产出文档本身**：和梳理文档一样有版本快照（编辑、AI 优化大纲都打快照）

**生成的 PPT**：每次完整"生成全部"产生一个版本，归档到左栏产出 tab：

```
📊 Q4 路演 PPT v3 (当前)
   2026-04-27 15:42 · 来自 Q4 路演 PPT 文档
   [下载] [查看预览]

📊 Q4 路演 PPT v2
   2026-04-26 17:20

📊 Q4 路演 PPT v1
   2026-04-25 11:00
```

逐页修改不产生新版本号——版本只对应"完整生成"事件。逐页修改累积在当前版本上。
